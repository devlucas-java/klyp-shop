# Contributing Guide

## 📌 Commit Pattern

Use Conventional Commits:

- feat: new feature
- fix: bug fix
- docs: documentation
- refactor: code improvement
- test: tests
- wip: work in progress

### ✅ Examples

feat: add product creation endpoint  
fix: resolve jwt validation bug  
docs: update README

❌ Avoid:

feat: stuff  
fix: things

---

## 🧼 Code Style

- Keep functions small
- Use clear naming
- Avoid unnecessary complexity

---

## 📦 Branch Strategy

- main → production
- develop → integration
- feature/* → new features

---

## ⚠️ Rules

- Always open Pull Request
- Review before merge
- Write meaningful commit messages
